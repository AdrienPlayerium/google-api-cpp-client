Handling Network Errors

This document describes the error handling model used within the client library's transport layer. The error handling model also covers HTTP redirects.

Basic concepts

The error handling policy is dictated by the HttpTransportErrorHandler class. The HttpRequest and HttpTransport delegate error handling to an instance of this class, which is made available through the HttpTransportOptions.

The HttpTransportErrorHandler is used to determine whether to retry a request and optionally how to modify request before retrying. The HttpRequest control flow calls into the error handler. All requests on a given HttpTransport instance use the error handler referenced by its HttpTransportOptions. Each transport instance has its own options so can use the same or different error handling policies.
